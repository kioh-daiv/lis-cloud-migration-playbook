# 医療DX シリーズ — オンプレ→クラウド 段階移行プレイブック

稼働中の臨床検査部門システム（LIS）を**止めずに** AWS クラウドへ移すための段階移行プレイブック。シングルページ HTML ガイド。

## 公開ページ

➡ **https://kioh-daiv.github.io/lis-cloud-migration-playbook/**

## 設計思想

- **横の証明（網羅性）**：① 移行ライフサイクル 5 フェーズ × ② AWS 7R の 2 軸で機械的に配置。対象/対象外を明示宣言。
- **縦の証明（深さ）**：各フェーズ・戦略を 要件 → 手段 → 検証 → 残リスク に分解。残リスクは「追加技術／運用／契約・SLA／受容」の 4 分類で引き取り。

## 構成

- スコープと網羅性の根拠
- §1 検査システム移行の三重制約（止められない／間違えられない／戻れない）
- §2 移行の 5 フェーズ（評価 → PoC → 並行稼働 → 切替 → 旧環境廃止）
- §3 AWS 7R を検査業務に当てはめる
- §4 データ移行（MGN / DMS / DataSync ＋ FHIR/JLAC ＋ リハーサル）
- §5 並行稼働・カットオーバー・ロールバック基準
- §6 落とし穴チェックリスト
- 出典（一次資料・公的情報）

## 位置づけ

医療DX シリーズの「深掘り編」。②④が「移行先の作り方」、本編が「そこへの渡り方」。移行時の記録の真正性は**データインテグリティ編**、移行後の継続運用は運用編（予定）に接続。

## 注意

2026 年 5 月時点の公的情報と AWS 公式ドキュメントに基づく一般整理。AWS のサービス名・仕様は改定される。移行計画は対象システムの実態・規制要件・ベンダー仕様に基づき、専門家の確認を前提に策定すること。
